Algorithms

Dijkstra's Algorithm (For Shortest Path Calculations)

Objective: Find the shortest path from one node to another in the network, ensuring fast access to key locations like hospitals in the event of a disaster.

Application: This algorithm can help determine the quickest routes between transport hubs, schools, and hospitals, ensuring that disaster response teams can reach critical areas quickly.

Prim's Algorithm (For Minimum Spanning Tree)

Objective: Ensure all the nodes in the network are connected in the most cost-efficient way, helping to reduce the overall risk of disaster response inefficiency.

Application: Prim’s algorithm can be used to design a robust road network that connects all important nodes (e.g., transport hubs, hospitals, and schools) at the minimum cost, ensuring accessibility even during emergencies.

Kruskal's Algorithm (For Minimum Spanning Tree)

Objective: Similar to Prim's algorithm, Kruskal's algorithm can also help connect all important nodes while minimizing the total connection cost.

Application: Kruskal's algorithm can be used to build a disaster response network that optimizes road connectivity between transport hubs, schools, and hospitals, ensuring that resources and teams can be deployed efficiently during emergencies.

Floyd-Warshall Algorithm (For All-Pairs Shortest Paths)

Objective: Calculate the shortest paths between every pair of nodes.

Application: This algorithm is useful if the platform needs to provide shortest-path information between all possible combinations of nodes (e.g., from any transport hub to any hospital or school).
